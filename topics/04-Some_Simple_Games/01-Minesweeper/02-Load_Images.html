Load Images

Download images

Create folder assets in your project and copy the following two images into that folder

Image tiles_transparent.png is tiles.png with a transparant background.

Screen setup

We want a grid of images that is centered both vertically and horizontally and is as large as possible given a fixed cell size. So in main.lua we first define the size:

1
local size = 64

then determine the maximum number of rows and cols possible based on display properties:

1
2
local rows = math.floor( display.contentHeight / size )
local cols = math.floor( display.contentWidth / size )

and the horizontal and vertical offsets required to center the grid:

1
2
local hoffset = (display.contentWidth - cols*size) / 2
local voffset = (display.contentHeight - rows*size) / 2

Next we hide the status bar, so we don't see the time while paying.

1
display.setStatusBar( display.HiddenStatusBar )

Importing images

In StarExplorer we used imagesheets, which are a image consisting of multiple smaller images. Since the images are of the same size, the image sheet setting can be constructed programmatically. 

1
2
3
4
5
6
7
local gameSheetOptions = { frames={} }
for row = 1, 2 do
    for col = 1, 8 do
        table.insert(gameSheetOptions.frames, {x=1+(col-1)*18, y=1+(row-1)*18, width=18, height=18 } )
    end
end
local gameSheet = graphics.newImageSheet("assets/tiles_transparent.png", gameSheetOptions)

At this point we can draw any of the images in gameSheet by specifying a number from 1 to 16. However, it would be nicer if we could refer to the images by name. There are multiple way to do this. Here I just create a second table, tiles, to use as lookup.

1
2
local tiles = { covered=1, covered_highlighted=2, uncovered=3, flower=4, flag=5, question=6, blank=7,
    n_1=9, n_2=10, n_3=11, n_4=12, n_5=13, n_6=14, n_7=15, n_8=16 }

Testing

To display a single tile at a single grid position we can now do the following:

-- test code (delete after use)
for k = 1, 16 do 
    local tile = display.newImageRect( gameSheet, k, size, size )
    tile.x = hoffset + (k-0.5) * size
    tile.y = voffset + (cols/2-0.5) * size
end
