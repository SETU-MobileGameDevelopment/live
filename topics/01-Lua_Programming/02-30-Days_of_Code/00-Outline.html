30 Days of Code Lua

Introduction

I have taken a selection of the problem set 30 Day of Code and created a simple lua based test interface that we can use on our own machines.
To use

  • Ensure that lua executable is in you system path.
  • Download the ZIP archive and extract contents.
  • The archive has structure

│   └── Day_20
├── soln              <- Location of your code (COPY OVER orig to start )
├── orig              <- Copy of original problem initial scripts (backup)
└── tests             <- Tests for each problem
    ├── Day_00
    ├── Day_02
    ├── Day_03
    . . .
  • Open your editor and open main lua file main.lua.
  • Rather than tests all of the scripts every time, set the command line parameter (Menu Project->Command line arguments...) to the problem number.
  • Complete the given problems.
  • Create ZIP archive with same structure as original, and upload using the link below. (due date to be decided on week 2, Wed session.)
